## 🚀 Product Launch Templates

### 1. Feature Launch (Classic "Now Live")

**Perfect for:** Announcing new features with excitement and driving immediate adoption

**Input fields:**
- **Feature Name** (required): The name of your new feature
- **User Benefit** (required): What problem it solves and why users should care
- **Example Use Case**: A specific scenario demonstrating value
- **CTA Link**: Link to try the feature

**Output format:**
```
🎉 Now Live: [Feature Name]

You asked, we built it.
→ [Short description of what it does]
→ [Why it matters / problem solved]

Try it today → [link]

#ProductUpdate #Launch #BuildInPublic
```

**Best practices:**
- Lead with excitement and urgency
- Clearly state the user benefit in the first few lines
- Include a strong call-to-action
- Use relevant hashtags for discoverability

---

### 2. Sneak Peek / Beta Invite

**Perfect for:** Building anticipation and creating FOMO for upcoming features

**Input fields:**
- **Feature Name** (required): Name of the upcoming feature
- **Value Proposition** (required): How it helps users
- **Limited Access Note**: Exclusivity details (e.g., "First 100 users")
- **Beta Link**: Sign-up URL (optional)

**Output format:**
```
👀 Sneak peek! We're testing something new: [Feature Name]

Designed to help you [value proposition].
We're inviting the first [X] users to try it early.

Drop a 🔥 or comment "beta" below to get access.

#BetaLaunch #StartupLife #AItools
```

**Best practices:**
- Create urgency with limited access
- Encourage engagement with clear CTAs
- Consider using emoji reactions as entry mechanism
- Follow up with selected participants

---

### 3. Problem → Solution → Proof

**Perfect for:** Demonstrating clear value by highlighting problems you solve

**Input fields:**
- **Problem Statement** (required): The pain point
- **Feature Name** (required): Your solution name
- **Solution Description** (required): How it solves the problem
- **Proof/Metric**: Evidence of effectiveness (optional)
- **Link**: Where to learn more

**Output format:**
```
❌ [Common pain/problem]
✅ Meet [Feature Name] — now in [Product Name].

[1-2 lines on how it fixes the issue]
[Optional proof: "Users cut posting time by 50%."]

Try it out: [link]
#ProductLaunch #BuildInPublic #FeatureDrop
```

**Best practices:**
- Start with a relatable problem
- Keep the solution explanation clear and concise
- Use metrics when available to add credibility
- Make the contrast (problem vs solution) visually clear

---

### 4. Built from Feedback

**Perfect for:** Showing you listen to your community and building loyalty

**Input fields:**
- **Feature Name** (required): What you built
- **Feedback Example**: Quote or paraphrase from users
- **Feature Improvement** (required): What it does and how it helps
- **Gratitude Message**: Thank your community (optional)

**Output format:**
```
💬 You asked for it — and we delivered!

[Feature Name] is here to [benefit].

Big thanks to everyone who shared feedback — your ideas help us build a better [product].

Keep it coming 👇
#CommunityDriven #FeatureUpdate
```

**Best practices:**
- Quote actual user feedback when possible
- Show genuine appreciation
- Encourage continued feedback
- Tag users who contributed (with permission)

## 🧠 Educational Templates

### 5. How It Works (Mini Guide)

**Perfect for:** Onboarding users and reducing complexity

**Input fields:**
- **Feature Name** (required): What you're explaining
- **Step 1-3** (required): Three clear steps
- **Pro Tip**: Advanced usage or bonus tip (optional)

**Output format:**
```
🚀 New in [Product Name]: [Feature Name]

Here's how to use it in 3 steps:
1️⃣ [Step 1]
2️⃣ [Step 2]
3️⃣ [Step 3]

Pro tip: [Extra bonus tip]

#HowTo #ProductUpdate #BuildInPublic
```

**Best practices:**
- Keep steps simple and actionable
- Use numbered emojis for visual clarity
- End with a pro tip for power users
- Consider adding a video or GIF

---

### 6. Before & After

**Perfect for:** Showcasing transformation and ROI

**Input fields:**
- **Old Way (Before)** (required): The previous pain point
- **Feature Name** (required): Your solution
- **New Way (After)** (required): The improved experience
- **Time/Metric Saved**: Quantifiable improvement (optional)
- **Link**: Where to try it

**Output format:**
```
Before: [Pain point, e.g. "Scheduling posts took hours"]
After: [Feature Name] automates it in seconds.

See the difference → [link]

#Automation #AItools #ProductUpdate
```

**Best practices:**
- Use concrete, measurable differences
- Keep before/after statements parallel in structure
- Include specific time or cost savings
- Visual comparison images work great here

---

### 7. Share Valuable Tips

**Perfect for:** Positioning yourself as an expert and providing value

**Input fields:**
- **Topic**: Your area of expertise
- **Tips**: Actionable advice to share

**Output format:**
```
[Number] tips for [topic]:

→ Tip 1
→ Tip 2
→ Tip 3
...

What would you add?

#Tips #[YourNiche] #BuildInPublic
```

**Best practices:**
- Make tips immediately actionable
- Use 3-7 tips (not too few, not overwhelming)
- End with a question to drive engagement
- Consider creating a thread for deeper dives

## Template Usage Guidelines

### Choosing the Right Template

1. **Define your goal first**
   - Announce something? → Product Launch
   - Teach something? → Educational
   - Connect personally? → Story & Narrative
   - Build community? → Engagement

2. **Know your audience**
   - B2B professionals → Professional tone
   - Startup community → Build in public angle
   - General audience → Casual and accessible

3. **Match platform norms**
   - LinkedIn: Professional, thought leadership
   - X/Twitter: Concise, conversational
   - Instagram: Visual, story-driven

### Customization Tips

- **Adapt the tone**: Templates are starting points, adjust language to match your brand voice
- **Add visuals**: Include relevant images, GIFs, or videos
- **Test variations**: Try different hooks and CTAs
- **Track performance**: Note which templates resonate with your audience

### Best Practices Across All Templates

1. **Be authentic**: Let your personality shine through
2. **Provide value**: Every post should educate, entertain, or inspire
3. **Include CTAs**: Guide your audience on next steps
4. **Use hashtags strategically**: 3-5 relevant tags for discoverability
5. **Engage with responses**: Reply to comments and build conversations
6. **Post consistently**: Regular posting builds audience and momentum
7. **A/B test**: Try variations to see what works best
8. **Monitor analytics**: Use data to refine your approach

### Common Mistakes to Avoid

❌ Being too sales-focused without providing value
❌ Using too many hashtags (looks spammy)
❌ Ignoring comments and engagement
❌ Copying templates verbatim without customization
❌ Posting at random times without strategy
❌ Not proofreading before publishing
❌ Overusing emojis or using them inappropriately

### Advanced Tips

1. **Create template variations**: Develop your own style within each category
2. **Build content series**: Use templates consistently for recurring content
3. **Cross-pollinate**: Combine elements from different templates
4. **Seasonal adaptations**: Adjust templates for holidays, events, trends
5. **Repurpose successful posts**: Use templates that worked before
